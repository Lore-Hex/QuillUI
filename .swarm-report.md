# GTK Describe Cycle Guard

## Root Cause

`gtkDescribeView` recursively follows `View.body` and reflected single-child wrappers. A representable host that wraps itself through `AnyView(Cyclic())` can recurse until Swift metadata lookup overflows the stack, producing a low-level crash instead of a GTK descriptor diagnostic.

## Changes

- Added a `nonisolated(unsafe)` GTK describe depth counter in `GTK4DescriptorTree.swift`.
- Past depth 200, the pass records `String(describing: type(of: view))` into an 8-entry ring buffer.
- Past depth 256, the pass reports `gtkDescribeView cycle: ...` with the last 8 type names and calls `fatalError`.
- Added internal `gtkDescribeDepthLimitExceededHandler` so tests can capture the diagnostic chain without aborting.
- Added Linux-only `GTKDescribeCycleGuardTests` with `Cyclic: View { AnyView(Cyclic()) }`.
