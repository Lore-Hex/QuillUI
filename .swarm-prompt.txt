ISSUE BODY:
**Roadmap item 2 — text rendering primitives.**

The current QuillPaint controls don't paint text — the chrome is rendered correctly but labels sit on top via the host. To make the controls truly self-contained AND match macOS metrics, QuillPaint needs a typography helper that emits text into a `PaintContext`.

### Context
- SF Pro is Apple-licensed and can't be redistributed
- Inter is the standard SF Pro substitute on Linux (Apache 2.0, very close metrics)
- macOS body label size: 13pt regular, weight 400, color matches `MacColors.controlText`
- macOS default button label: 13pt regular, color `MacColors.controlText` for normal, `defaultButtonText` for default
